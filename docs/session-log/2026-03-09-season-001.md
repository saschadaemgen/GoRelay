## Session 1 (March 9, 2026) - Research and Architecture

### SMP Server Analysis

The simplexmq Haskell server at github.com/simplex-chat/simplexmq was analyzed in depth. Key findings:

Concurrency model uses three threads per client connection (reader, writer, processor), mapping directly to Go's goroutine model.

Queue model uses unidirectional simplex queues with exactly one sender and one recipient per queue, identified by random 24-byte IDs. Only one connection can subscribe to a queue at a time.

The server re-encrypts messages before delivery using NaCl crypto_box with a DH-derived shared secret. Transport uses fixed 16,384-byte padded blocks after TLS handshake.

No existing Go implementation of SMP existed anywhere. GoRelay would be the first.

### Go Server Patterns

Go's runtime uses epoll/kqueue internally. At 3 goroutines per connection, 10,000 concurrent clients consume approximately 60MB of stack memory. The practical limit is file descriptors, not goroutines.

Go's crypto/tls natively supports custom ALPN and non-HTTP TLS servers. The block transport uses io.ReadFull for exact reads - plain conn.Read may return partial data.

### Technology Stack Selection

Go 1.24+ was selected for single binary compilation, goroutine concurrency, and stdlib ML-KEM support. BadgerDB was chosen over BBolt (100x better write throughput), Pebble (no native TTL), and SQLite (single writer, no native TTL). koanf for configuration (Viper lowercases keys). log/slog for stdlib logging. Prometheus for metrics. golang.org/x/time/rate for rate limiting.

### Security Improvements Over SMP

Six concrete improvements were identified and adopted as architecture decisions:

1. Noise Protocol replaces TLS for GRP - no CA dependency, no cipher negotiation, built-in identity hiding
2. Mandatory post-quantum cryptography - hybrid X25519 + ML-KEM-768, no fallback
3. Two-hop relay routing as default - sender-recipient unlinkability
4. Server-generated cover traffic - Poisson-distributed dummy messages
5. Automatic queue rotation - queue IDs rotate every 24-72 hours
6. Aggressive message retention - delete on ACK, 48h default TTL, 7d hard max

### The No-Groups Decision

Group messaging forces cryptographic compromises in every existing system. Signal uses Sender Keys (weaker forward secrecy). Matrix uses Megolm (weaker forward secrecy). By eliminating groups entirely, every message has full Double Ratchet with Perfect Forward Secrecy and Post-Compromise Security. No exceptions.

### Dual-Protocol Architecture

The key architectural insight: GoRelay IS the bridge between SMP and GRP. Port 5223 speaks SMP for SimpleX compatibility. Port 7443 speaks GRP with Noise transport, post-quantum crypto, and cover traffic. Both frontends share the same QueueStore and SubscriptionHub. A message sent via SMP is deliverable to a GRP subscriber and vice versa.

### Noise Protocol Deep Dive

Noise_IK (WireGuard's pattern) selected as primary - 1 round trip, 0-RTT encrypted payloads. Noise_XX as fallback for first contact. Go library: flynn/noise v1.1.0 (BSD-3). For post-quantum extension: katzenpost/noise with Hybrid Forward Secrecy.

### Post-Quantum Cryptography

FIPS 203 (ML-KEM) finalized August 2024. Go 1.24 stdlib includes FIPS-validated, Trail-of-Bits-audited ML-KEM. GRP/1 fixed cipher suite: X25519 + ML-KEM-768 hybrid, HKDF-SHA-256, ChaCha20-Poly1305, BLAKE2s, Ed25519. No version negotiation, no fallback.

### Legal Analysis

GDPR Article 5(1)(c) (data minimization) and 5(1)(e) (storage limitation) favor zero-knowledge relays. German Vorratsdatenspeicherung suspended since 2017 (CJEU SpaceNet ruling 2022). If metadata is never generated, freeze orders have nothing to preserve.
